Back button behaviour is tab specific to behave like chrome

Behaviour: [NAVIGATE ANY HISTORY BACK, CLOSE ACTIVE TAB+MINIMISE APP]. Triggered by:

  • Open link from whatsapp

Behaviour: [NAVIGATE ANY HISTORY BACK, CLOSE ACTIVE TAB+SHOW SPEED DIAL, MINIMISE APP. After returning, there should be ghost tab where u left it]. Triggered by:

  • From speed dial, click fave/history item/etc
  • Switch tab
  • Close all tabs

Behaviour: [NAVIGATE ANY HISTORY BACK, CLOSE ACTIVE TAB, SHOW PREV TAB]. Triggered by:

  • Press + tab button from nav bar then open a link or address
  • Open clicked/selected link/image in new tab
  • Expand preview window to full tab

Behaviour: [BACK TO CURRENT TAB USING TEMP FLAG]. Triggered by:

  • Selecting address hint or speed dial list

Behaviour: [SMOOTH SCROLL BACK TO TAB, REMOVE UNUSED ON SETTLE]. Triggered by:

  • Pressing +tab button then pressing back

Other use cases:

  • Pressing +tab button then clicking on an existing tab link in the tab list. Swipe and check no unused tabs remain
